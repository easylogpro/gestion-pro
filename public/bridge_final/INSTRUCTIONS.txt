========================================
   BRIDGE ACCESS - INSTRUCTIONS
========================================

1. PRÉREQUIS
============
- Python installé (https://www.python.org/downloads/)
- Cocher "Add Python to PATH" lors de l'installation
- Microsoft Access ou Access Database Engine installé

2. INSTALLATION
===============
a) Copier tout le dossier bridge_final sur votre PC
b) Vérifier le chemin de la base Access dans bridge_server.py :
   DATABASE_PATH = r"C:\Users\Administrateur\Desktop\CONTRAT 2022.accdb"
   
c) Le fichier contacts.json contient les mails/tels des techniciens et STT

3. LANCEMENT
============
a) Double-cliquer sur LANCER_BRIDGE.bat
b) Le serveur démarre sur http://localhost:5000
c) Ouvrir votre application (app.html)
d) Cliquer sur "Connexion Access (Bridge)"

4. INTÉGRATION DANS APP.HTML
============================
Ajouter ces 3 lignes dans app.html :

Ligne ~4237 (avant </style>) :
    </style>
    <link rel="stylesheet" href="bridge.css">
</head>

Ligne ~11750 (avant </body>) :
    </script>
    <script src="bridge.js"></script>
</body>

Ligne ~4359 (modifier le onclick) :
    <div class="import-option access" onclick="connectBridge()" id="bridgeConnectOption">

5. FICHIERS
===========
- bridge_server.py : Serveur Python (API)
- bridge.js : Module JavaScript (connexion)
- bridge.css : Styles (indicateur status)
- contacts.json : Mails/tels techniciens et STT
- LANCER_BRIDGE.bat : Script de lancement

6. ENDPOINTS API
================
- /api/ssi      → Liste des contrats en cours 2008
- /api/dsf      → Liste en cours DSF 2008
- /api/dsf/cloture → DSF à clôturer
- /api/dsf/planif  → DSF à planifier
- /api/dsf/resp    → DSF à voir RESP
- /api/devis    → Liste generale travaux
- /api/travaux  → Liste travaux en cours
- /api/reco     → Reconditionnement 2008
- /api/tech     → Techniciens (+ contacts.json)
- /api/stt      → Sous-traitants (+ contacts.json)

7. DÉPANNAGE
============
Erreur "Python non trouvé" :
→ Réinstaller Python avec "Add to PATH" coché

Erreur "pyodbc" :
→ Installer Access Database Engine 64-bit

Erreur connexion Access :
→ Vérifier le chemin DATABASE_PATH dans bridge_server.py

Les données ne s'affichent pas :
→ Ouvrir la console (F12) et vérifier les erreurs
→ Tester http://localhost:5000/api/test dans le navigateur
